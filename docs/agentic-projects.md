<h2 id="english">🇬🇧 English</h2>

This page indexes the projects where the LLM has a main role.

The list is organized by the role played by the language model, not by product category. A Unity, 3D, music or image project can appear here when the LLM is part of the actual workflow.

---

## Project index

| Project | LLM-centered role | Main review angle |
| --- | --- | --- |
| **Codex Model Orchestrator** | Task routing, tool calls, readable runs and decision surfaces. | Can the run be understood after the agent finishes? |
| **Agent Unity** | Scene reading, visual review, editor assistance and issue preparation. | Can the agent help identify and prepare useful Unity work? |
| **CodexToUnity** | Turns a request into a Unity-oriented task path with target, manifest, checks and review notes. | Is the Unity handoff clear before any import or integration step? |
| **Mob’ia Android** | Mobile surface for LLM-assisted profiles, jobs, artifacts and review states. | Can a user follow generation work from a mobile surface? |
| **Mob’ia Unity Extension** | Unity-side link between Mob’ia jobs, artifacts and LLM-assisted review. | Can artifact state be understood in scene context? |
| **Mob’ia / ccomf-unity** | Product layer around ComfyUI-style jobs and client-readable states. | Does raw generation become a readable product workflow? |
| **Splat to Assets** | LLM-guided organization of splat, multiview and 3D candidate processing. | Can generated 3D material become a structured asset candidate? |
| **LocalAssetFactory** | Manifest, validation and review loop for generated asset candidates. | Are generated candidates reviewable before being treated as assets? |
| **uwdevst_hub** | Synth preset generation, audition, comparison and evaluation with LLM support. | Can sound candidates be evaluated instead of only generated? |
| **Codex Image Génial** | Image editing workflow where Codex helps describe, script, compare and review edits. | Can image changes be driven and reviewed from the discussion? |
| **MCP / Apps SDK surfaces** | Controlled tool actions, confirmations, status views and result cards. | Are read/write actions, confirmations and results clear to a user? |

## Reading rule

This repository should stay focused on the LLM role.

When a project is also about Unity, 3D, music or image editing, the technical product details can live in another showcase. Here, the useful question is:

```text
What does the LLM read, prepare, generate, check or help decide?
```

Cette page indexe les projets dans lesquels le LLM occupe un rôle principal.

La liste est organisée par le rôle du modèle de langage, pas par catégorie produit. Un projet Unity, 3D, musique ou image peut apparaître ici lorsque le LLM fait partie du vrai workflow.

## Index des projets

| Projet | Rôle centré LLM | Angle principal de revue |
| --- | --- | --- |
| **Codex Model Orchestrator** | Routage de tâches, appels d’outils, runs lisibles et surfaces de décision. | Le run reste-t-il compréhensible après le passage de l’agent ? |
| **Agent Unity** | Lecture de scène, revue visuelle, assistance éditeur et préparation de problèmes. | L’agent aide-t-il à identifier et préparer un travail Unity utile ? |
| **CodexToUnity** | Transforme une demande en chemin de tâche orienté Unity avec cible, manifest, contrôles et notes de revue. | Le handoff Unity est-il clair avant import ou intégration ? |
| **Mob’ia Android** | Surface mobile pour profils, jobs, artefacts et états de revue assistés par LLM. | Un utilisateur peut-il suivre la génération depuis une surface mobile ? |
| **Mob’ia Unity Extension** | Lien côté Unity entre jobs Mob’ia, artefacts et revue assistée par LLM. | L’état d’un artefact est-il compréhensible dans le contexte scène ? |
| **Mob’ia / ccomf-unity** | Couche produit autour des jobs type ComfyUI et états lisibles côté client. | La génération brute devient-elle un workflow produit lisible ? |
| **Splat to Assets** | Organisation guidée par LLM du traitement splat, multiview et candidats 3D. | La matière 3D générée devient-elle un candidat asset structuré ? |
| **LocalAssetFactory** | Boucle manifest, validation et revue pour candidats assets générés. | Les candidats générés sont-ils reviewables avant d’être traités comme assets ? |
| **uwdevst_hub** | Génération, écoute, comparaison et évaluation de presets synthé avec support LLM. | Les candidats sonores peuvent-ils être évalués plutôt que seulement générés ? |
| **Codex Image Génial** | Workflow d’édition d’image où Codex aide à décrire, scripter, comparer et revoir les edits. | Les changements image peuvent-ils être pilotés et revus depuis la discussion ? |
| **Surfaces MCP / Apps SDK** | Actions outil contrôlées, confirmations, vues de statut et cartes de résultat. | Les actions read/write, confirmations et résultats sont-ils clairs pour l’utilisateur ? |
