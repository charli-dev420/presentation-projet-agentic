<h2 id="english">🇬🇧 English</h2>

This page indexes the projects where the LLM has a main role.

The list is organized by the role played by the language model, not by product category. A Unity, music or image project can appear here when the LLM is part of the actual workflow.

---

## Project index

| Project | LLM-centered role |
| --- | --- |
| **Codex Model Orchestrator** | Task routing, tool calls, readable runs and decision surfaces. |
| **Agent Unity** | Scene reading, visual review, editor assistance and issue preparation. |
| **CodexToUnity** | Turns a request into a Unity-oriented task path with target, manifest, checks and review notes. |
| **Mob’ia Android** | Mobile surface for LLM-assisted profiles, jobs, artifacts and review states. |
| **Mob’ia Unity Extension** | Unity-side link between Mob’ia jobs, artifacts and LLM-assisted review. |
| **Mob’ia / ccomf-unity** | Product layer around ComfyUI-style jobs and client-readable states. |
| **Splat to Assets** | LLM-guided organization of splat, multiview and 3D candidate processing. |
| **LocalAssetFactory** | Manifest, validation and review loop for generated asset candidates. |
| **uwdevst_hub** | Synth preset generation, audition, comparison and evaluation with LLM support. |
| **Codex Image Génial** | Image editing workflow where Codex helps describe, script, compare and review edits. |
| **MCP / Apps SDK surfaces** | Controlled tool actions, confirmations, status views and result cards. |

---

## Reading rule

This repository should stay focused on the LLM role.

When a project is also about Unity, 3D, music or image editing, the technical product details can live in another showcase. Here, the useful question is:

```text
What does the LLM read, prepare, generate, check or help decide?
```

## Index des projets

| Projet | Rôle centré LLM |
| --- | --- |
| **Codex Model Orchestrator** | Routage de tâches, appels d’outils, runs lisibles et surfaces de décision. |
| **Agent Unity** | Lecture de scène, revue visuelle, assistance éditeur et préparation de problèmes. |
| **CodexToUnity** | Transforme une demande en chemin de tâche orienté Unity avec cible, manifest, contrôles et notes de revue. |
| **Mob’ia Android** | Surface mobile pour profils, jobs, artefacts et états de revue assistés par LLM. |
| **Mob’ia Unity Extension** | Lien côté Unity entre jobs Mob’ia, artefacts et revue assistée par LLM. |
| **Mob’ia / ccomf-unity** | Couche produit autour des jobs type ComfyUI et états lisibles côté client. |
| **Splat to Assets** | Organisation guidée par LLM du traitement splat, multiview et candidats 3D. |
| **LocalAssetFactory** | Boucle manifest, validation et revue pour candidats assets générés. |
| **uwdevst_hub** | Génération, écoute, comparaison et évaluation de presets synthé avec support LLM. |
| **Codex Image Génial** | Workflow d’édition d’image où Codex aide à décrire, scripter, comparer et revoir les edits. |
| **Surfaces MCP / Apps SDK** | Actions outil contrôlées, confirmations, vues de statut et cartes de résultat. |
