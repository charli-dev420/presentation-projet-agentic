<h2 id="english">🇬🇧 English</h2>

This page indexes the projects where the LLM has a main role.

The list is organized by the role played by the language model, not by product category. A Unity, 3D, music or image project can appear here when the LLM is part of the actual workflow.

---

## Project index

| Project | LLM-centered role | Review angle |
| --- | --- | --- |
| **Codex Model Orchestrator** | Task routing, tool calls, readable runs and decision surfaces. | Can a run be understood after it happened? |
| **Agent Unity** | Scene reading, visual review, editor assistance and issue preparation. | Can the LLM help inspect a Unity scene without hiding the review step? |
| **CodexToUnity** | Turns a request into a Unity-oriented task path with target, manifest, checks and review notes. | Can a request become a reviewable Unity handoff? |
| **Mob’ia Android** | Mobile surface for LLM-assisted profiles, jobs, artifacts and review states. | Can a user follow generation work from a mobile surface? |
| **Mob’ia Unity Extension** | Unity-side link between Mob’ia jobs, artifacts and LLM-assisted review. | Can job state connect clearly to scene-level review? |
| **Mob’ia / ccomf-unity** | Product layer around ComfyUI-style jobs and client-readable states. | Can a raw generation workflow become product-readable? |
| **Splat to Assets** | LLM-guided organization of splat, multiview and 3D candidate processing. | Can visual candidates be turned into asset steps? |
| **LocalAssetFactory** | Manifest, validation and review loop for generated asset candidates. | Can generated candidates be checked before being accepted? |
| **uwdevst_hub** | Synth preset generation, audition, comparison and evaluation with LLM support. | Can sound candidates be reviewed instead of only generated? |
| **Codex Image Génial** | Image editing workflow where Codex helps describe, script, compare and review edits. | Can edits be directed and reviewed from the discussion? |
| **MCP / Apps SDK surfaces** | Controlled tool actions, confirmations, status views and result cards. | Can tool state stay readable to the person using it? |

---

## Reading rule

This repository should stay focused on the LLM role.

When a project is also about Unity, 3D, music or image editing, the technical product details can live in another showcase. Here, the useful question is:

```text
What does the LLM read, prepare, generate, check or help decide?
```

## Index des projets

| Projet | Rôle centré LLM | Angle de revue |
