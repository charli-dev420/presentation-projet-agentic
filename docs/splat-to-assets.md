<h2 id="english">🇬🇧 English</h2>

Splat to Assets is a pipeline direction for turning splats, multiview material or generated 3D candidates into reusable assets.

It can also belong in a 3D showcase. Here, the focus is the LLM role: organizing the candidate, naming expectations, preparing checks and explaining what can become a usable asset.

---

## LLM role

The LLM can help:

- describe what the candidate represents;
- prepare conversion or cleanup steps;
- structure metadata;
- list quality checks;
- summarize what is reusable, broken or uncertain.

---

## Useful workflow

```text
splat / multiview / candidate → interpretation → asset plan → checks → reusable package
```

The goal is to move from generated visual material toward assets that can be reviewed, packaged and reused.

---

## Review signals

| Signal | Meaning |
| --- | --- |
| Candidate is named | The source material has an intended role. |
| Conversion path is visible | The steps toward asset packaging are not vague. |
| Quality checks exist | Geometry, texture, scale, orientation or reuse limits can be reviewed. |
| Uncertainty is kept | Broken or incomplete parts are not hidden. |

---

## Public boundary

This page presents the LLM-guided workflow angle. It does not expose private datasets, private model outputs, internal conversion scripts or unreleased asset packs.

<h2 id="francais">🇫🇷 Français</h2>

Splat to Assets est une direction pipeline pour transformer des splats, vues multiples ou candidats 3D générés en assets réutilisables.

Le projet peut aussi appartenir à une vitrine 3D. Ici, le focus est le rôle du LLM : organiser le candidat, nommer les attentes, préparer les contrôles et expliquer ce qui peut devenir un asset utilisable.

---

## Rôle du LLM

Le LLM peut aider à :

- décrire ce que représente le candidat ;
- préparer les étapes de conversion ou nettoyage ;
- structurer les métadonnées ;
- lister les contrôles qualité ;
- résumer ce qui est réutilisable, cassé ou incertain.

---

## Workflow utile

```text
splat / multiview / candidat → interprétation → plan asset → contrôles → package réutilisable
```

Le but est de passer d’une matière visuelle générée vers des assets qui peuvent être revus, packagés et réutilisés.

---

## Signaux de revue

| Signal | Sens |
| --- | --- |
| Le candidat est nommé | La matière source a un rôle attendu. |
| Le chemin de conversion est visible | Les étapes vers le packaging asset ne sont pas vagues. |
| Les contrôles qualité existent | Géométrie, texture, échelle, orientation ou limites de réutilisation peuvent être revues. |
| L’incertitude est gardée | Les parties cassées ou incomplètes ne sont pas cachées. |

---

## Limite publique

Cette page présente l’angle workflow guidé par LLM. Elle n’expose pas les datasets privés, sorties de modèles privées, scripts internes de conversion ou packs assets non publiés.
