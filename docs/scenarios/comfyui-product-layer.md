# Scenario: ComfyUI Product Layer

## Flow

1. Mob'ia / ccomf-unity receives a product-level request.
2. A profile selects the intended generation path.
3. A job is created, monitored, and converted into a reviewable artifact.
4. Unity, Android, or web clients display the result status.
5. A user accepts, revises, or archives the artifact.

## Public Output

A product journey card with profile, job status, artifact type, and review decision. No backend route, token, storage detail, Docker/run config, workflow, generated private image, or provider output is published.
