<h2 id="english">🇬🇧 English</h2>

Use this page for a short first review of the repository.

---

## Five-minute path

1. Read the README to understand the LLM-centered positioning.
2. Read [`agentic-projects.md`](agentic-projects.md) to see the project map.
3. Read [`llm-roles.md`](llm-roles.md) to understand what the LLM actually does.
4. Pick one project page, not all of them.
5. Check whether the role, workflow and public boundary are clear.

---

## What to check first

| Check | Good sign |
| --- | --- |
| Positioning | The repo is about LLM-centered workflows, not a general Unity/3D/music dump. |
| Project map | Each project has a distinct role. |
| LLM role | The model reads, plans, edits, evaluates or helps decide. |
| Human control | The workflow keeps review and final decision visible. |
| Boundary | Private material is not exposed. |

---

## Review questions

A useful first review answers:

- Is the agentic angle clear?
- Is the LLM role understandable?
- Does any project sound overpromised?
- Is the documentation path simple enough?
- Which project should be expanded first?

## Parcours en cinq minutes

1. Lire le README pour comprendre le positionnement centré LLM.
2. Lire [`agentic-projects.md`](agentic-projects.md) pour voir la carte projet.
3. Lire [`llm-roles.md`](llm-roles.md) pour comprendre ce que fait réellement le LLM.
4. Choisir une page projet, pas toutes.
5. Vérifier si le rôle, le workflow et la limite publique sont clairs.

---

## Ce qu’il faut vérifier d’abord

| Vérification | Bon signal |
| --- | --- |
| Positionnement | Le repo parle de workflows centrés LLM, pas d’un dump Unity/3D/musique général. |
| Carte projet | Chaque projet a un rôle distinct. |
| Rôle LLM | Le modèle lit, planifie, édite, évalue ou aide à décider. |
| Contrôle humain | Le workflow garde la revue et la décision finale visibles. |
| Limite | La matière privée n’est pas exposée. |

---

## Questions de revue

Une première revue utile répond à :

- L’angle agentic est-il clair ?
- Le rôle du LLM est-il compréhensible ?
- Un projet semble-t-il survendu ?
- Le parcours documentaire est-il assez simple ?
- Quel projet devrait être développé en premier ?
