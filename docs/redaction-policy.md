# Publication Hygiene / Hygiene de publication

[EN](#english) | [FR](#francais)

## English

### Before Adding Evidence

- Replace prompts, traces, commands, routes, endpoints, run ids, local paths, user names, provider outputs, and logs with product-language summaries.
- Redraw architecture instead of screenshotting private dashboards.
- Use synthetic examples or cleared sample data.
- Keep model internals, workflow internals, storage details, tokens, keys, Docker/run configs, and generated private assets out of the repo.
- Add the target reader: user, collaborator, funder, recruiter, or technical reviewer.

### Image Rule

Images must be diagrams, synthetic examples, or cleared captures. Any capture must hide endpoint, route, local path, run id, prompt, workflow, model, log, generated private asset, user name, and account metadata.

## Francais

### Avant D'Ajouter Une Preuve

- Remplacer prompts, traces, commandes, routes, endpoints, run ids, chemins locaux, noms utilisateur, sorties provider et logs par des syntheses produit.
- Redessiner l'architecture au lieu de capturer des dashboards internes.
- Utiliser des exemples synthetiques ou donnees d'exemple validees.
- Garder internes modele, internes workflow, details stockage, tokens, cles, configs Docker/run et assets prives generes hors repo.
- Ajouter le lecteur cible: utilisateur, collaborateur, financeur, recruteur ou reviewer technique.

### Regle Image

Les images doivent etre diagrammes, exemples synthetiques ou captures validees. Toute capture doit masquer endpoint, route, chemin local, run id, prompt, workflow, modele, log, asset prive genere, nom utilisateur et metadata compte.
