# Redaction Policy / Politique de redaction

This repository may include rewritten summaries and diagrams. It must not publish raw operational material.

## Before Adding Any Proof

- Replace prompts, traces, commands, routes, endpoints, run ids, local paths, user names, provider outputs, and logs with product-language summaries.
- Redraw architecture instead of screenshotting private dashboards.
- Use synthetic examples or cleared demo data.
- Keep model names, workflow internals, storage details, tokens, keys, Docker/run configs, and generated private assets out of the repo.
- Label evidence level: `L0`, `L1`, `L2`, `L3`, `L4`, or `L5`.
- Add a target reader: user, buyer, collaborator, funder, recruiter, or technical reviewer.

## Image Rule

Images must be diagrams, synthetic examples, or fully redacted captures. Any capture must hide endpoint, route, local path, run id, prompt, workflow, model, log, generated private asset, user name, and account metadata.
