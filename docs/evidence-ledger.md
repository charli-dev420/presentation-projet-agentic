# Evidence Ledger / Registre de preuves

[EN](#english) | [FR](#francais)

## English

This ledger keeps claims tied to a public validation signal.

| ID | Surface | Signal | What it supports | Reference |
| --- | --- | --- | --- | --- |
| AI-EV-01 | Codex Model Orchestrator | Readable run model | Goal, scope, tool path, checks, and human decision can be described as one product flow. | [Orchestrator](projects/codex-model-orchestrator.md) |
| AI-EV-02 | MCP / App SDK | Reader-facing state model | Tool status, actions, evidence, and decisions can be shown as product surfaces. | [Ecosystem map](ecosystem-map.md) |
| AI-EV-03 | CodexToUnity | Unity handoff model | A generated candidate can be prepared for manifest, import, and review. | [CodexToUnity](projects/codextounity.md) |
| AI-EV-04 | Mob'ia / ccomf-unity | Job/artifact lifecycle | ComfyUI work can be framed as profiles, jobs, artifacts, clients, and review states. | [Mob'ia](projects/mobia-ccomf-unity.md) |
| AI-EV-05 | LocalAssetFactory | Local asset validation loop | Generation output can be checked through normalization, Unity import, and scene usefulness. | [LocalAssetFactory](projects/local-asset-factory.md) |
| AI-EV-06 | QA | Decision-oriented checks | A run can be evaluated through scope, artifact, checks, evidence, and decision. | [QA validation](qa-validation.md) |

| AI-EV-05 | LocalAssetFactory | Boucle validation asset locale | Une sortie de generation peut etre controlee via normalisation, import Unity et utilite scene. | [LocalAssetFactory](projects/local-asset-factory.md) |
